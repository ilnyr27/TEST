#### Тест «Слабые стороны и слепые пятна» (Shadow)

**Файлы:** `src/lib/test-engine/shadow-data.ts`, `src/lib/scoring/shadow-scorer.ts`

42 вопроса, 6 измерений, scale 1-5 («Совсем не про меня» → «Очень точно»):

| Код | Измерение | Цвет | Что измеряет |
|-----|-----------|------|-------------|
| PROC | Прокрастинация | Янтарный | Откладывание, избегание старта |
| ANXI | Тревога и гиперконтроль | Индиго | Беспокойство, контроль, неопределённость |
| CONF | Избегание конфликтов | Изумрудный | Трудности с «нет», подавление несогласия |
| PERF | Перфекционизм | Фиолетовый | Страх ошибок, самокритика, блокировка старта |
| SELF | Самосаботаж | Красный | Синдром самозванца, отказ от возможностей |
| IMPU | Импульсивность | Оранжевый | Реактивность, нетерпение, импульсивные решения |

Каждое измерение даёт развёрнутое описание (раздельно для высокого/низкого балла) + конкретный практический совет. Высокий балл = черта активно проявляется, без осуждения.

Зарегистрирован в `scorer-registry.ts` и `test-registry.ts`, задеплоен.

#### Тест «Глубокий Big Five» (Deep Big Five)

**Файлы:** `src/lib/test-engine/deep-big-five-data.ts`, `src/lib/scoring/deep-big-five-scorer.ts`

80 вопросов, IPIP-NEO adapted. 5 измерений × 16 вопросов каждое. В отличие от BFI-44, вопросы поведенческие («Когда что-то идёт не по плану, меня охватывает раздражение»), а не декларативные («Я эмоционально стабилен»).

Каждое измерение покрывает 6 граней (фасетов) NEO PI-R:

| Измерение | Фасеты |
|-----------|--------|
| N (Нейротизм) | Тревожность, Вспыльчивость, Уныние, Застенчивость, Импульсивность, Уязвимость |
| E (Экстраверсия) | Теплота, Общительность, Напористость, Активность, Поиск острых ощущений, Позитивные эмоции |
| O (Открытость) | Фантазия, Эстетика, Чувства, Открытость действиям, Интеллект, Гибкость убеждений |
| A (Дружелюбие) | Доверие, Прямота, Альтруизм, Мягкость в конфликтах, Скромность, Сочувствие |
| C (Добросовестность) | Компетентность, Порядок, Обязательность, Целеустремлённость, Самодисциплина, Обдуманность |

Обратное кодирование (`reverse: true`) применяется к ~30% вопросов.  
Оригинальный Big Five (44 вопроса) оставлен как быстрый вариант.

- `src/app/api/subscription/cancel/route.ts` — NEW: устанавливает `subscription_cancelled_at`
- `src/app/api/subscription/renew/route.ts` — NEW: крон-эндпоинт для автопродления
- `src/hooks/usePlan.ts` — возвращает `subscriptionPlan`, `isSubActive`, `subscriptionCancelled`
- `src/app/[locale]/(main)/pricing/page.tsx` — карточки Про/Макс сверху

**Планы подписки:**

| План | Цена | Сессий/мес | Сообщ/сессию |
|------|------|-----------|-------------|
| Про | 499 ₽/мес | 20 | 40 |
| Макс | 990 ₽/мес | 999 (∞) | 80 |

Оба плана включают `has_report = true` (полный отчёт в подарок).

**Как работает:**
1. Первая оплата: `save_payment_method: true` → ЮКасса сохраняет карту
2. Webhook `payment.succeeded` → записывает `subscription_plan`, `subscription_expires_at` (+1 месяц), `subscription_payment_method_id`, добавляет сессии
3. Крон на сервере: `0 9 * * *` → вызывает `/api/subscription/renew?secret=<CRON_SECRET>`
4. Крон заряжает сохранённым методом за 2 дня до истечения → webhook продлевает ещё на месяц
5. Отмена: POST `/api/subscription/cancel` → `subscription_cancelled_at = now()` → крон не продлит, доступ до конца периода
6. Если продление не прошло → `subscription_cancelled_at` ставится автоматически (защита от зависания)

**БД миграция:**
```sql
ALTER TABLE user_plans 
  ADD COLUMN subscription_plan TEXT,
  ADD COLUMN subscription_expires_at TIMESTAMPTZ,
  ADD COLUMN subscription_payment_method_id TEXT,
  ADD COLUMN subscription_cancelled_at TIMESTAMPTZ;
```

## Архитектурные решения

**Почему self-hosted Supabase:** облачный supabase.co заблокирован в РФ через Cloudflare.

**Почему не Vercel:** 152-ФЗ — данные должны быть в РФ. Деплой на домашний сервер Ubuntu + Docker.

**Почему npm-зеркало:** npmjs.com недоступен из РФ по IPv4. В Docker: `registry.npmmirror.com`.

**DeepSeek и 152-ФЗ:** сообщения уходят в Китай — трансграничная передача по ст.12. Обязательно в политике конфиденциальности.

**Тест-движок:** данные — TypeScript-файлы (не БД), scoring — классы (`Scorer` интерфейс), registry pattern (`test-registry.ts`, `scorer-registry.ts`). Результаты — localStorage + `results-store.ts`.

**Монетизация (актуально на 2026-07-19):**

| Продукт | Цена | Что даёт |
|---------|------|----------|
| Бесплатно | 0 ₽ | 1 сессия (12 сообщ.) + 1 ИИ-анализ теста |
| Про подписка | 499 ₽/мес | 20 сессий × 40 сообщ. + полный отчёт |
| Макс подписка | 990 ₽/мес | ∞ сессий × 80 сообщ. + полный отчёт |
| Сессии разово | 99–369 ₽ | 3/7/15 сессий × 40 сообщ. |
| Полный отчёт | 299 ₽ | Разовый доступ к ИИ-отчёту |
