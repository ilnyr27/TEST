## Исправленные

### БАГ-011: Share-карточка — поле `subject` вместо `name`

**Симптом:** TypeScript build error.  
**Где:** `src/lib/share/generate-card.ts`  
**Причина:** `ShareRadarPoint` объявляли поле `subject`, но реальный `result.radarData` использует `name`.  
**Решение:** Переименовали `subject` → `name` в интерфейсе и в функции `drawRadar`.  
**Дата:** 2026-07-12

---

### БАГ-010: Claude кнопка в тарифах — можно было купить нерабочие сессии

**Симптом:** На /pricing была кнопка покупки Claude-сессий, но Claude отключён в коде.  
**Решение:** Убрали кнопку Claude из pricing.

---

### БАГ-009: Цены в оферте не совпадали с реальными ценами

**Симптом:** Оферта описывала "3 кредита — 149₽", реально "3 сессии — 99₽".  
**Риск:** По ЗОЗПП — основание для претензий.  
**Решение:** Синхронизировали цены и терминологию в `offer/page.tsx`, `terms/page.tsx`.

---

### БАГ-008: Двойная локаль в URL — `/ru/ru/login`

**Симптом:** Кнопка "Зарегистрироваться" на /pricing вела на `/ru/ru/login` (404).  
**Причина:** `<Link href={\`/${locale}/login\`}>` + Link из next-intl сам добавляет локаль.  
**Решение:** `<Link href="/signup">` — без префикса.  
**Паттерн:** Никогда не включать `/${locale}/` в href при использовании Link из next-intl.

---

### БАГ-007: Полный отчёт обрезался на 5–6 разделе

**Симптом:** PDF обрывался на середине.  
**Где:** `src/lib/ai/deepseek-provider.ts`, метод `analyze()`  
**Причина:** `max_tokens: 2000`. Полный 8-разделный отчёт требует 3500–5000 токенов.  
**Решение:** `max_tokens: 6000` + проверка `finish_reason === "length"` → throw вместо сохранения обрезанного.

---

### БАГ-006: Docker кэш скрывал ESLint ошибки

**Симптом:** Build падал внезапно после изменения несвязанного файла.  
**Причина:** Docker кэшировал COPY/RUN шаги — старые ошибки не всплывали пока кэш не сбрасывался.  
**Урок:** После ESLint-suppress комментариев делать `docker build --no-cache` для проверки.

---

### БАГ-005: `git pull` не подтягивал коммиты на сервере

**Симптом:** "Already up to date", но новых файлов нет.  
**Причина:** Не сделали `git push` с локальной машины.  
**Порядок деплоя:** commit → push → SSH pull → docker compose up --build.

### БАГ-002: ESLint `react-hooks/set-state-in-effect` ломал build

**Где:** `src/hooks/usePlan.ts`  
**Причина:** Правило запрещает синхронный setState в useEffect.  
**Решение:** `// eslint-disable-next-line react-hooks/set-state-in-effect` перед каждым вызовом.  
**Применяется:** `usePlan.ts`, `coach/page.tsx`, `full-report/page.tsx`

### БАГ-001: Race condition — платный пользователь видит freemium

**Симптом:** Пользователь купил отчёт, заходит на /full-report — видит заблюренные разделы.  
**Где:** `src/hooks/usePlan.ts`  
**Причина:** `planLoading` оставался `false` пока Supabase грузил данные. `hasReport = null?.has_report ?? false = false`.  
**Решение:** `setLoading(true)` перед fetch в ветке `user !== null`.

## Полезные паттерны

**next-intl Link:**
- ❌ `href={\`/${locale}/path\`}` → дублирует локаль
- ✅ `href="/path"` → next-intl добавит локаль автоматически
- При `window.location.href` — нужно явно: `` `/${locale}/path` ``

**setState в useEffect:**  
ESLint блокирует синхронный setState в начале useEffect. Обход: `// eslint-disable-next-line react-hooks/set-state-in-effect`.
